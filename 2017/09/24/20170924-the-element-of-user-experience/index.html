《用户体验要素》读后感

发表于 2017-09-24 | 分类于 读书笔记

一本小书，带你领略 Ajax 之父——Jesse James Gsrrett 眼中的用户体验。

《用户体验要素》基于“战略层，范围层，结构层，框架层和表现层”五个层面的基础架构来讲解用户体验。作者谈到整个软件产品设计过程中各个层面的一些注意事项，同时，为了避免混淆，还定义了一些业内便于交流的术语。另外，文中讲述了很多开发过程中一些看起来习以为常但实质是有理论依据的案例。整本书排版大方，内容简洁，结构分明。下面总结自己看书过程中的一点摘录和感悟。

产品设计 VS 用户体验设计

“外形服从于功能”，这种观点对于内部运作(用户不可见的部分)是适用的。但是，对于产品直接面对用户的那些部分，正确的产品形态绝不是由“功能”所决定的，而是由“用户自身的心理感受和行为”来决定的。

产品设计以及开发初级阶段，难免会有专注于功能而选择性忽略界面设计的嫌疑。这个时候在每一步的开发过程中就应该自己花心思来美化界面，通过一种帮助人们理解和接受的方式来呈现。感觉这个也需要一定平日积累，经常的情况是：“我知道这个地方很丑，但是不知道怎么改进。”

转化率

一个常用的投资收益的度量标准是转化率。通过跟踪弄有百分之多少的用户被你“转化”到了下一个步骤，就能衡量你的网站在达到“商业目的”方面的效率有多高。

刚好最近在做留存分析的界面，看到书中提到的转化率，不免心中有点点融会贯通的感觉。为了提高转化率，用户体验可以围绕“帮助人们工作得更快”和“减少他们犯错的几率”这两个方面进行改进。通过转化率可以量化用户体验对于用户留存的贡献值。

概念模型

用户对于“交互组件将怎样工作”的观点成为概念模型。

书中列举了两个例子：购物车和未改版前的西南航空公司网站。电子商务网站的购物车的图标能很形象地让用户明白它的作用，与之相反的是西南航空的网站：首页是一张客户服务的书桌的图片，书桌上一边堆着简介的小册子，另一边放着电话。因为过分运用概念模型，反而适得其反。脑补一下，你肯定觉得网站做成这样简直不可思议。
现实和概念，就像生活和艺术，概念源于生活，但要经过抽象后高于生活。

命名原则 nomenclature

好的命名原则可以辅助结构层让用户更好地找到他们想走的路。

  • 受控词典 controlled vocabulary

    受控词典是网站使用的一套标准语言。与用户谈话并了解他们的沟通方式，是开发出一个让用户感到自然的命名原则系统的最有效方式。

  • 类词词典 thesaurus

    类词词典会提供常用的、但未纳入该网站标准用语的词汇以供选择，你可添加内部专用术语、速写语、俚语或缩写词等与其相对应的词汇进行补充。

  • 元数据 metadata
    元数据是“关于信息的信息”，即以一种结构化的方式来描述内容的信息。构建受控词典和类词词典对于建立包含元数据的系统特别有用。

视觉词典 visual vocabulary

作者创建的图解网站结构的系统——视觉词典。详情可见——视觉词典文档。

用户习惯与布局

习惯使我们可以把反射作用应用到不同的环境中。

文中举例，有的制造商试着设计一些不符合“三行四列”的按钮布局的电话，结果每一次打电话时大约需要花3秒钟左右。一个不符合习惯的设计使得一件“下意识就能完成的事情”变成了“难以忍受的缓慢”。对于用户而言，这3秒钟不仅仅是时间的浪费，还有耐心的消磨和被挫折感占据的沮丧。在没有充分理由和明显益处时，不要轻易尝试强行通过界面改变用户习惯。

关于文档

当然，所有这些文档是大量的工作，但是有一个很好的理由值得去做：随着时间的推移，我们的决策原因会逐渐从记忆中消失。那些“在某种特殊环境下用来解决某个具体问题的临时决策”和那些“为了形成将来的设计工作的基础而有意识地做出来的决策”会混杂在一起。

总结

由于对于软件整体缺乏认知，读后感并没有按照书中的五个层面分别展开。只是总结了看书过程中觉得比较过瘾的部分。在开发过程中可以结合这些概念和观点，致力于创造出更好的用户体验。
